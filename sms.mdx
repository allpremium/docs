---
title: "Sms"
description: "Send a message via SMS, Telegram, or WhatsApp."
---

The request first checks that your account has sufficient balance. If successful, the message is queued for delivery. A [callback URL](/callback) is required to receive the final status; you can also poll the [status](/payments/status) endpoint.

**POST** `/v1/bus/send/{service}`

---

## Services

| Service | Description |
|---------|-------------|
| `sms` | SMS |
| `telegram` | Telegram |
| `whatsapp` | WhatsApp |

---

## Request

<RequestExample>
```bash cURL
curl -X POST https://api.tupay.africa/v1/bus/send/sms \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer {token}' \
  -d '{
    "sender":   "TEST",
    "to":       "254722000000",
    "message":  "This is a test",
    "schedule": "now",
    "unique":   "yes"
  }'
```
</RequestExample>

<ParamField body="sender" type="string" required>
  Sender ID — [email us](mailto:hello@allpremium.co.ke) to register one.
</ParamField>

<ParamField body="to" type="string | array" required>
  Recipient phone number(s) — up to 1,000.
</ParamField>

<ParamField body="message" type="string" required>
  Message body.
</ParamField>

<ParamField body="schedule" type="string">
  Send timing: `now` or `repeat`.
</ParamField>

<ParamField body="options" type="string">
  Repeat options — required when `schedule` is `repeat`.
</ParamField>

<ParamField body="unique" type="string">
  Filter duplicates: `yes` or `no`.
</ParamField>
