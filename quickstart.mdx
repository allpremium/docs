## 1. Prerequisites

- A verified Tupay business account — [activate your account](https://tupay.africa).
- Your **API credentials** (uuid + key) from the Tupay dashboard.
- A tool to make HTTPS requests (e.g. `curl`, Postman, or an HTTP client in your language).

All requests must be made over **HTTPS** and include a valid `Authorization` header.
See [Authorization](/prerequisites/authorization) for full details.

```
Base URL:      https://api.tupay.africa
```

---

## 2. Get your API key

1. Sign in to the [Tupay dashboard](https://tupay.africa/dashboard).
2. Navigate to **Developers / API keys**.
3. Create a key pair and copy:
   - **Consumer UUID** — identifies your integration.
   - **Consumer Key** — your API key.
4. Store them securely; treat them like a password.

You will exchange them for a short-lived bearer token:

```
Authorization: Bearer {access_token}
```

See the [Authorization](/prerequisites/authorization) guide for the token exchange flow.

---

## 3. Make your first request

The fastest way to confirm your integration is to retrieve your current balance.

```bash
curl -X GET https://api.tupay.africa/v1/core/balance \
  -H "Authorization: Bearer {token}"
```

```json
{
  "status":   20,
  "amount":   100.0,
  "currency": "KES"
}
```

- `status: 20` means success — see [Status Codes](/status).
- `amount` is your available balance in the given `currency`.

---

## 4. Accept your first payment

1. **Create a payment** with the customer details and amount.
2. The customer receives an **STK push** (or equivalent prompt) to complete payment.
3. **Receive a callback** from Tupay with the final status.
4. Optionally **poll the status** endpoint for an immediate update.

Follow the [Accept a Payment](/payments/accept) guide for a full walkthrough.

---

## 5. Handle callbacks and errors

Tupay sends **HTTP callbacks** to your server whenever a transaction changes state.

- Implement a callback endpoint as described in [Callbacks](/callback).
- Use the `status` field from [Status Codes](/status) to decide how to update your system:

| Status | Meaning |
|--------|---------|
| `20` | Success — payment complete |
| `0`, `3`, `31`, `100`, `101` | Still pending — wait for callback |
| Any other | Failed — review and handle |
