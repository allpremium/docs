---
title: "Identity"
description: "Verify a customer's identity via KYC document check."
---

The request first checks that your account has sufficient balance. If successful, a verification order is placed. A [callback URL](/callback) is required to receive the final result; you can also poll the [status](/payments/status) endpoint.

**POST** `/v1/pay/verify/{service}`

---

## Services

| Service | Description |
|---------|-------------|
| `kyc` | Know Your Customer document verification |

---

## Request

<RequestExample>
```bash cURL
curl -X POST https://api.tupay.africa/v1/pay/verify/kyc \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer {token}' \
  -d '{
    "documentType":   "ID",
    "documentNumber": "1234",
    "countryCode":    "KE",
    "firstName":      "John",
    "lastName":       "Maina",
    "dateOfBirth":    "2000-01-01",
    "reference":      "ref1234"
  }'
```
</RequestExample>

<ParamField body="documentType" type="string" required>
  Document type: `ID`, `PASSPORT`, or `ALIENID`.
</ParamField>

<ParamField body="documentNumber" type="string" required>
  Document number.
</ParamField>

<ParamField body="countryCode" type="string" required>
  ISO country code e.g. `KE`, `RW`.
</ParamField>

<ParamField body="firstName" type="string" required>
  Customer first name.
</ParamField>

<ParamField body="lastName" type="string" required>
  Customer last name.
</ParamField>

<ParamField body="dateOfBirth" type="string" required>
  Date of birth in `yyyy-MM-dd` format.
</ParamField>

<ParamField body="reference" type="string">
  Your internal reference.
</ParamField>

---

## Response

<ResponseExample>
```json Response
{

  "reference": "ref1234",
```

<ResponseField name="status" type="number">

</ResponseField>
