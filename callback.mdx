---
title: "Callbacks"
description: "Receive real-time transaction status updates via HTTP webhooks."
---

Tupay posts a callback to your configured URL when a transaction reaches a final state. Set your callback URL in [Settings](https://tupay.africa/dashboard/settings).

<Info>
  Your callback endpoint must return `HTTP 200` to acknowledge receipt. Tupay will retry failed deliveries automatically.
</Info>

---

## Callback payload

<ResponseExample>
```json Callback body
{
  "id":        "ABCDEFG250101",
  "reference": "ref1234",
  "status":    20,
  "message":   "Success",
  "balance":   100,
  "currency":  "KES",
  "comm":      5
}
```
</ResponseExample>

<ResponseField name="id" type="string">
  Tupay transaction ID.
</ResponseField>

<ResponseField name="reference" type="string">
  Your internal reference.
</ResponseField>

<ResponseField name="status" type="number">
  Transactional status code. See [Status Codes](/status).
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable status description.
</ResponseField>

<ResponseField name="currency" type="string">
  Currency code (e.g. `KES`).
</ResponseField>

<ResponseField name="balance" type="number">
  Your account balance after the transaction.
</ResponseField>

<ResponseField name="comm" type="number">
  Commission earned on the transaction (optional).
</ResponseField>

<ResponseField name="identity" type="object">
  Identity verification result — only present on [Identity](/payments/identity) requests.
</ResponseField>

---

## Verifying callbacks

Callbacks are sent as HTTP `POST` requests to your configured URL. Verify that incoming requests originate from Tupay by checking the source IP against Tupay's published IP range, or by validating a shared secret if configured.
