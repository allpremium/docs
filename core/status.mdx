---
title: "Retrieve a transaction"
description: "Check the current status of any transaction."
---

**GET** `/v1/status/{id}`

Returns the current status of any transaction regardless of type. You can also use the resource-scoped endpoints which return the same response:

| Endpoint | Description |
|----------|-------------|
| `GET /v1/airtime/{id}` | [Retrieve an airtime order](/airtime/order#retrieve-an-airtime-order) |
| `GET /v1/bills/{id}` | [Retrieve a bill payment](/bills/order#retrieve-a-bill-payment) |
| `GET /v1/payouts/{id}` | [Retrieve a payout](/payments/payouts#retrieve-a-payout) |

<Info>
  For real-time updates, prefer [webhooks](/webhooks) instead of polling.
</Info>

<RequestExample>
```bash cURL
curl -X GET https://api.tupay.africa/v1/status/ABCDEFG250101 \
  -H 'Authorization: Bearer {token}'
```
</RequestExample>

### Path parameter

| Parameter | Description |
|-----------|-------------|
| `id` | Transaction ID or your internal reference |

---

<ResponseExample>
```json Response
{
  "id":        "ABCDEFG250101",
  "reference": "ref1234",
  "status":    20,
  "message":   "Success"
}
```
</ResponseExample>

<ResponseField name="id" type="string">
  Tupay transaction ID.
</ResponseField>

<ResponseField name="reference" type="string">
  Your internal reference.
</ResponseField>

<ResponseField name="status" type="number">
  See [Status Codes](/status).
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable status.
</ResponseField>
